# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is an **educational lab** (Day 22) for building a production-grade RAG pipeline instrumented with LangSmith observability, Prompt Hub versioned prompt management, RAGAS automated evaluation, and Guardrails AI output safety. The repo currently contains pseudocode skeletons that students implement.

## Environment

- Python 3.10+
- Dependencies: `pip install -r requirements.txt`
- Environment variables loaded from `.env` via `python-dotenv`
- `.env` must never be committed — it contains API keys

Key packages and their versions:
- `langchain >= 0.3.0`, `langchain-openai >= 0.3.0`, `langchain-community >= 0.3.0`
- `langsmith >= 0.2.0`
- `ragas >= 0.4.0` (NOT 0.3.x — metrics import from `ragas.metrics`, not `ragas.metrics.collections`)
- `guardrails-ai >= 0.5.0` (0.10.x recommended)
- `faiss-cpu >= 1.7.0`
- `numpy >= 1.25.0`

## Architecture

Four independent scripts that build on each other sequentially:

1. **`01_langsmith_rag_pipeline.py`** — Loads a knowledge base text file, chunks it with `RecursiveCharacterTextSplitter`, indexes chunks in FAISS (`FAISS.from_texts`), builds an LCEL RAG chain (`retriever → prompt → LLM → StrOutputParser`), then decorates the query function with `@traceable` to send traces to LangSmith.

2. **`02_prompt_hub_ab_routing.py`** — Defines two `ChatPromptTemplate` instances (V1: concise, V2: structured). Pushes both to LangSmith Prompt Hub via `client.push_prompt()`, pulls them back via `client.pull_prompt()`. Routes queries deterministically using `hashlib.md5(request_id) % 2` — even hash → V1, odd hash → V2.

3. **`03_ragas_evaluation.py`** — Runs all 50 QA pairs through both prompt versions, collects answers and retrieved contexts (as `list[str]` — not joined). Builds `EvaluationDataset` from `SingleTurnSample` objects. Evaluates with 4 RAGAS metrics: `faithfulness`, `answer_relevancy`, `context_recall`, `context_precision`. Must achieve faithfulness ≥ 0.8 for at least one version.

4. **`04_guardrails_validator.py`** — Two custom validators:
   - `PIIDetector` — Regex-based detection of email, phone, SSN, credit card. Uses `PassResult(value_override=...)` with redacted text. Decorated with `@register_validator`.
   - `JSONFormatter` — Validates JSON parseability. Auto-repairs: strips markdown fences, converts single quotes to double quotes, removes trailing commas. Falls back to `FailResult` if irreparable.

## Critical Patterns & Gotchas

### LangSmith tracing
- `os.environ["LANGCHAIN_TRACING_V2"] = "true"` must be set **before** importing LangChain
- Use `@traceable(name="...")` from `langsmith` (not `langchain`)

### RAGAS 0.4.x
- Import metrics directly: `from ragas.metrics import faithfulness` (NOT `from ragas.metrics.collections`)
- `evaluate()` returns lists per sample per metric — use `numpy.mean()` to average
- Pass `llm=` and `embeddings=` to `evaluate()`, not to metric constructors
- `SingleTurnSample.retrieved_contexts` must be a `list[str]`, not a joined string

### Guardrails AI
- `on_fail=OnFailAction.FIX` goes in the **validator constructor**, not `Guard.use()`: `Guard().use(PIIDetector(on_fail=OnFailAction.FIX))`
- `Guard.validate(text)` is the main entry point; returns `ValidationOutcome` with `.validation_passed` and `.validated_output`

### Deterministic A/B routing
- Use `int(hashlib.md5(request_id.encode()).hexdigest(), 16) % 2` — same `request_id` always maps to same version
- Not random — this is graded

### Chunking baseline
- `RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)` is the suggested starting point
- If faithfulness < 0.8, try smaller chunks or adjust chunk_overlap
