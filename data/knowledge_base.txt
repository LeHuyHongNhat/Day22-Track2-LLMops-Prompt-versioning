Machine Learning Fundamentals

Supervised learning uses labeled training data where each example has an input-output pair. The model learns to map inputs to outputs. Common algorithms include linear regression, decision trees, and neural networks. Unsupervised learning works with unlabeled data to find hidden patterns. Clustering algorithms like K-means and DBSCAN group similar data points. Dimensionality reduction techniques like PCA reduce feature space while preserving variance. Reinforcement learning trains agents through trial and error using reward signals. The agent takes actions in an environment and learns optimal policies through cumulative reward maximization.

Overfitting and Model Evaluation

Overfitting occurs when a machine learning model learns the training data too well, capturing noise and random fluctuations rather than the underlying pattern. This leads to poor generalization on unseen data. Signs of overfitting include very low training error but high validation error. The bias-variance tradeoff describes the relationship between model complexity and error. High bias (underfitting) means the model is too simple to capture the underlying pattern. High variance (overfitting) means the model is too complex and sensitive to small fluctuations in the training data. The optimal model balances both to minimize total error. Regularization techniques like L1 (Lasso) and L2 (Ridge) add penalty terms to the loss function to discourage overly complex models. L1 regularization can drive weights to zero for feature selection. L2 regularization shrinks weights toward zero but keeps all features. Cross-validation splits data into multiple folds, training on some and validating on others, to estimate model performance more reliably than a single train-test split. Common approaches include k-fold and stratified k-fold.

Neural Networks and Deep Learning

Backpropagation is the algorithm that computes gradients of the loss function with respect to each weight in a neural network by applying the chain rule of calculus. Gradients flow backward from the output layer through hidden layers to update weights via gradient descent. Convolutional Neural Networks (CNNs) are specialized for processing grid-like data such as images. They use convolutional layers with learnable filters that slide across the input to detect spatial patterns like edges and textures. Pooling layers reduce spatial dimensions through downsampling operations like max pooling or average pooling, decreasing computational complexity while retaining the most important features. LSTM (Long Short-Term Memory) networks address the vanishing gradient problem in recurrent neural networks by using gating mechanisms. The forget gate, input gate, and output gate control information flow through the network, allowing LSTMs to learn long-range dependencies. Common activation functions include ReLU (Rectified Linear Unit) which outputs max(0,x), sigmoid which squashes values to (0,1), and tanh which outputs values in (-1,1).

Transformer Architecture and NLP

The transformer architecture was introduced in the paper "Attention Is All You Need" (2017). It uses self-attention mechanisms that allow each token to attend to all other tokens in the sequence, processing entire sequences in parallel rather than sequentially. This parallelization enables much faster training compared to RNNs. Self-attention computes query, key, and value matrices from input embeddings and calculates attention weights as the softmax of query-key dot products. Multi-head attention runs multiple attention operations in parallel, allowing the model to focus on different representation subspaces. Word embeddings represent words as dense vectors in a continuous vector space where semantically similar words are geometrically close. Popular embedding methods include Word2Vec, GloVe, and contextual embeddings from models like BERT. Transfer learning in NLP involves pre-training large language models on massive text corpora, then fine-tuning them on specific downstream tasks with smaller labeled datasets. This approach has revolutionized NLP by leveraging knowledge learned from billions of words.

BERT (Bidirectional Encoder Representations from Transformers) uses bidirectional transformer training, looking at both left and right context simultaneously. It is pre-trained using masked language modeling (predicting masked tokens) and next sentence prediction (determining if two sentences follow each other). GPT (Generative Pre-trained Transformer) uses autoregressive training, predicting the next token given all previous tokens. It is trained on massive text datasets and can generate coherent long-form text. GPT-4 supports up to 128K tokens of context length, allowing it to process very long documents. Instruction tuning involves fine-tuning pre-trained LLMs on datasets of instruction-response pairs to improve alignment with human intent and make models better at following directions. RLHF (Reinforcement Learning from Human Feedback) uses human preference rankings to train a reward model, then uses reinforcement learning to align the LLM to be helpful, harmless, and honest. Chain-of-thought prompting encourages LLMs to show their reasoning step by step, significantly improving performance on complex reasoning tasks.

Retrieval-Augmented Generation (RAG)

Retrieval-Augmented Generation (RAG) combines generative LLMs with retrieval from external knowledge bases to produce grounded, up-to-date answers. Instead of relying solely on the model's internal knowledge, RAG retrieves relevant documents and conditions the LLM's response on that retrieved context. The main components of a RAG pipeline are a retriever that searches a document store for relevant passages and a generator (LLM) that produces answers from the query and retrieved passages. Dense retrieval uses neural embeddings to encode both queries and documents into the same vector space, determining relevance by computing cosine similarity between query and document embeddings. This is more semantic and flexible compared to sparse keyword matching like BM25.

Chunking strategy is critical in RAG because it affects retrieval precision and context window fit. Options include fixed-size chunking (splitting by token count), semantic chunking (splitting at natural boundaries like paragraphs), and hierarchical chunking (using multiple levels of granularity). Larger chunks provide more context but may include irrelevant information. Smaller chunks are more precise but may miss necessary context. A common baseline is chunk_size=500 with chunk_overlap=50. Advanced RAG techniques beyond basic retrieval include re-ranking (using a more powerful model to reorder retrieved documents), query expansion (generating multiple query variations), HyDE (Hypothetical Document Embeddings — generating a hypothetical answer and using it as the query), and iterative retrieval for multi-hop questions that require chaining multiple retrieval steps.

Vector Databases and Similarity Search

Vector databases store and query high-dimensional vector embeddings for fast similarity search. They are essential infrastructure for RAG pipelines, semantic search, and recommendation systems. FAISS (Facebook AI Similarity Search) is a library for efficient similarity search and clustering of dense vectors. It supports both exact search (IndexFlatL2 for brute-force L2 distance) and approximate nearest neighbor algorithms (IndexIVFPQ for compressed vectors). Text embeddings convert text into numerical vectors where semantically similar texts produce geometrically close vectors in the embedding space. Embedding models like text-embedding-3-small map text to fixed-length vectors, typically 1536 dimensions. HNSW (Hierarchical Navigable Small World) builds a hierarchical graph structure for logarithmic-complexity approximate nearest neighbor search. It constructs multiple layers where top layers have long-range edges and bottom layers have short-range edges, enabling fast traversal. Hybrid search combines dense vector search with sparse keyword search (BM25) using Reciprocal Rank Fusion to merge results. This captures both semantic similarity and exact keyword matching for better retrieval quality.

LangChain Ecosystem

LangChain is an open-source framework for building LLM applications with abstractions for chains, agents, and data connections. It provides standardized interfaces for LLMs, vector stores, document loaders, and tools. LangChain Expression Language (LCEL) is a declarative way to compose chains using the pipe operator (|). It supports streaming, async execution, batching, and automatic parallelization. A typical LCEL chain looks like: prompt | llm | output_parser. LangGraph extends LangChain for building stateful multi-actor applications as directed graphs, supporting cycles and conditional edges. It is well-suited for agent-based systems where the flow depends on intermediate results. LangChain supports several memory types: ConversationBufferMemory (stores all messages), ConversationSummaryMemory (summarizes older messages), ConversationWindowMemory (keeps the last k messages), and vector store memory (retrieves relevant past messages). LangChain retrievers fetch relevant documents from a data source given a query. They provide a common interface over various backends including vector stores (FAISS, Chroma), BM25, and custom retrieval logic.

LangSmith Platform

LangSmith is a platform for debugging, testing, evaluating, and monitoring LLM applications through automatic tracing. It captures detailed traces of every LLM call, showing inputs, outputs, latency, token usage, and errors for every component in a chain. The LangSmith Prompt Hub is a repository for storing, versioning, and sharing prompt templates. It supports A/B testing by serving different prompt versions and comparing their performance, as well as rollback to previous versions. LangSmith helps monitor production LLM applications by providing latency percentiles, error rates, token cost tracking, and feedback annotations. Developers can attach feedback (thumbs up/down, scores) to traces. LangSmith datasets are used for systematic evaluation, containing example inputs and expected outputs. They enable comparing model versions or prompt changes by running the same evaluation set and measuring performance differences through online or offline evaluation modes.

RAGAS Evaluation Framework

RAGAS is an open-source framework for evaluating RAG pipelines using LLM-based reference-free metrics. It provides multiple metrics to assess different aspects of RAG quality. RAGAS computes faithfulness by extracting atomic claims from the generated answer and checking whether each claim can be inferred from the retrieved context. A faithful answer is grounded in the provided documents without hallucination. Answer relevancy measures how well the answer addresses the original question. It works by generating synthetic questions from the answer and computing the cosine similarity between embeddings of the original question and the generated questions. Context recall measures how well the retrieved context covers the information needed to answer the question, evaluated against a ground truth reference answer. Context precision measures whether relevant documents are ranked higher than irrelevant ones in the retrieved results. RAGAS evaluation requires user queries, generated answers, retrieved contexts (as a list of individual passages, not a single joined string), and optionally reference answers for recall metrics.

Guardrails AI and Safety

Guardrails AI is an open-source framework for adding validation and safety checks to LLM outputs with configurable on-fail actions. It uses a Validator pattern where custom validators can detect, fix, or reject problematic outputs. Validators are registered with @register_validator and implement a validate() method that returns PassResult or FailResult. OnFailAction.FIX replaces the output with a safe alternative instead of raising an error. PII (Personally Identifiable Information) includes data like email addresses, phone numbers, social security numbers, and credit card numbers. Detecting and redacting PII in LLM responses is critical because exposing it can violate GDPR and HIPAA privacy regulations. Structured output validation ensures that LLM responses conform to expected schemas such as valid JSON. This can involve auto-repairing common formatting issues like markdown code fences, single quotes, or trailing commas. Constitutional AI is a technique where a model is given principles to follow and uses self-critique to revise its responses, enabling safer and more aligned outputs. Common AI safety concerns with LLMs include hallucination (generating false information), toxicity (offensive content), bias (unfair treatment of groups), PII leakage (exposing private data), and jailbreaking attacks (bypassing safety restrictions).
